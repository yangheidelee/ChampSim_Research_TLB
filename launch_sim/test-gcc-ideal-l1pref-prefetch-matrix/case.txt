## 配置矩阵

脚本会生成并运行以下配置：

```text
pref-berti-pythia
  ideal mode: none
  L1D: berti
  L2C: pythia

pref-berti-no
  ideal mode: none
  L1D: berti
  L2C: no

pref-berti-ip_stride
  ideal mode: none
  L1D: berti
  L2C: ip_stride

pref-no-pythia
  ideal mode: none
  L1D: no
  L2C: pythia

ideal-l1pref-berti-pythia
  ideal mode: l1pref
  L1D: berti
  L2C: pythia

ideal-l1pref-berti-no
  ideal mode: l1pref
  L1D: berti
  L2C: no

ideal-l1pref-berti-ip_stride
  ideal mode: l1pref
  L1D: berti
  L2C: ip_stride

ideal-l1pref-no-pythia
  ideal mode: l1pref
  L1D: no
  L2C: pythia

ideal-all-berti-pythia
  ideal mode: all
  L1D: berti
  L2C: pythia

ideal-all-berti-no
  ideal mode: all
  L1D: berti
  L2C: no

ideal-all-berti-ip_stride
  ideal mode: all
  L1D: berti
  L2C: ip_stride

ideal-all-no-pythia
  ideal mode: all
  L1D: no
  L2C: pythia
```

这 12 项构成当前验证矩阵：

```text
pref / ideal_l1pref / ideal_all
  x
Berti+Pythia / Berti+NoL2 / Berti+IPStride / NoL1D+Pythia
```

Berti+Pythia 这组用于复现已有现象：

```text
pref Berti+Pythia
ideal_l1pref Berti+Pythia
ideal_all Berti+Pythia
```

Berti+NoL2、Berti+IPStride 和 NoL1D+Pythia 的非 ideal pref baseline 用于分别和同一 prefetcher 组合下的 ideal_l1pref/ideal_all 做公平对比：

```text
pref Berti+NoL2
ideal_l1pref Berti+NoL2
ideal_all Berti+NoL2

pref Berti+IPStride
ideal_l1pref Berti+IPStride
ideal_all Berti+IPStride

pref NoL1D+Pythia
ideal_l1pref NoL1D+Pythia
ideal_all NoL1D+Pythia
```

`ideal_all` 下的多组用于拆分 L1D Berti 与不同 L2C prefetcher 的交互：

```text
ideal_all Berti+NoL2
ideal_all Berti+IPStride
ideal_all NoL1D+Pythia
ideal_all Berti+Pythia
```

## 输出

result log：

```text
results/test-gcc-ideal-l1pref-prefetch-matrix/<config-tag>/*.log
```

汇总 CSV：

```text
csv_figure/test-gcc-ideal-l1pref-prefetch-matrix/gcc_ideal_prefetch_matrix_summary.csv
csv_figure/test-gcc-ideal-l1pref-prefetch-matrix/gcc_ideal_prefetch_matrix_delta_vs_pref.csv
csv_figure/test-gcc-ideal-l1pref-prefetch-matrix/gcc_ideal_prefetch_matrix_delta_vs_same_pref.csv
```

其中 `delta_vs_same_pref.csv` 是最重要的对比表：每个 ideal 结果都会和同一组 prefetcher 的非 ideal pref baseline 相比。

汇总字段包括：

```text
IPC / cycles
STLB total miss / STLB L1D prefetch miss / STLB demand miss
L1D LOAD miss / MSHR merge / unique-to-L2C
L2C LOAD access / hit / miss / MSHR merge / unique-to-LLC
L1D/L2C prefetch issued / useful / useless / late / accuracy
LLC LOAD access / hit / miss
L1D/L2C/LLC average miss latency
```
