Purpose
=======

This folder tests whether the paper-style L1D/L2C data-cache prefetchers
are responsible for changing the IPC upper bound of ideal STLB all-hit
modeling.

It only compares:

1. pref
2. ideal-all

The baseline for speedup is pref. The old comparison data is read from:

/home/zcq/git_prj/champsim_ideal_stlb_ipc_upperbound/csv_figure/1core-spec0617_gapligra-idealTLB-select-trace-compare_jsonnew/select_trace

No existing launch script or source file is modified by this case.

Run Commands
============

Build, run, and summarize:

MAX_PARALLEL=5 N_WARM=20 N_SIM=50 \
./launch_sim/test-ideal-stlb-paper-prefetcher-only-pref-vs-all/run_test.sh all

Only build:

./launch_sim/test-ideal-stlb-paper-prefetcher-only-pref-vs-all/run_test.sh build

Only run existing binaries:

MAX_PARALLEL=5 N_WARM=20 N_SIM=50 \
./launch_sim/test-ideal-stlb-paper-prefetcher-only-pref-vs-all/run_test.sh run

Only regenerate CSV after logs exist:

./launch_sim/test-ideal-stlb-paper-prefetcher-only-pref-vs-all/run_test.sh summary

Config Changes In base_config.json
==================================

Base file:

/home/zcq/git_prj/champsim_ideal_stlb_ipc_upperbound/launch_sim/1core-spec0617_gapligra-idealTLB-select-trace-compare_jsonnew/ideal-all-workload-sweep/1C.fullBW.stlb.stats.json

Only these two fields are changed:

- L1D.prefetcher: berti -> next_line
- L2C.prefetcher: pythia -> ip_stride

Everything else intentionally remains the same as the jsonnew base file,
including:

- CPU core width and ROB.
- L1I/L1D/L2C/LLC size, associativity, latency, MSHR, and replacement.
- DRAM timing and capacity.
- TLB, PTW, PSC, and page table configuration.
