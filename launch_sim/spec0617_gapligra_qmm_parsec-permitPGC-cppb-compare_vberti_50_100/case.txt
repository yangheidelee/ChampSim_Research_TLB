# Permit PGC vs Permit PGC + CP-PB

工作目录：

cd /home/zcq/git_prj/ChampSim

脚本入口：

launch_sim/spec0617_gapligra_qmm_parsec-permitPGC-cppb-compare_vberti_50_100/run_tlb_select_compare.sh

实验语义：

1. permit PGC baseline：
   - 使用 pref-workload-sweep
   - L1D prefetcher 为 vberti，允许跨页预取
   - 不开启 STLB CP-PB

2. permit PGC + CP-PB：
   - 使用 pref-cppb-workload-sweep
   - JSON 配置和 permit PGC 一致
   - 运行 binary 时额外带上 --enable-stlb-cp-pb

3. 对比指标：
   - IPC speedup：permit PGC + CP-PB / permit PGC
   - STLB demand MPKI reduction：以 permit PGC 为 baseline，看 CP-PB 后 demand STLB MPKI 的减少百分比

trace 选择：

1. 默认 SELECT_SET=stlb：
   - 使用 nopref 下 STLB MPKI > 1.0 的 selected trace
   - JSON 路径：
     launch_sim/spec0617_gapligra_qmm_parsec-permitPGC-cppb-compare_vberti_50_100/selected_traces/stlb_mpki_gt_1.0_selected_traces.json
   - csv_figure 输出目录：
     csv_figure/spec0617_gapligra_qmm_parsec-permitPGC-cppb-compare_vberti_50_100/stlb_mpki_gt_1_select_trace

2. SELECT_SET=llc-spec：
   - 使用 nopref 下 LLC MPKI > 1.0 的 trace，但只保留 SPEC06/SPEC17
   - JSON 路径：
     launch_sim/spec0617_gapligra_qmm_parsec-permitPGC-cppb-compare_vberti_50_100/selected_traces/llc_mpki_gt_1.0_spec_selected_traces.json
   - csv_figure 输出目录：
     csv_figure/spec0617_gapligra_qmm_parsec-permitPGC-cppb-compare_vberti_50_100/spec_llc_mpki_gt_1_select_trace

常用命令：

N_WARM=50 N_SIM=100 \
