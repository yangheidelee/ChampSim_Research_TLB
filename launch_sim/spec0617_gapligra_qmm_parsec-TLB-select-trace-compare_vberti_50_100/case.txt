# 1core SPEC06 + SPEC17 + GAP + Ligra + QMM + PARSEC + XSBench TLB select-trace compare

工作目录：

cd /home/zcq/git_prj/ChampSim

本脚本文件夹只处理当前实验：

launch_sim/spec0617_gapligra_qmm_parsec-TLB-select-trace-compare_vberti_50_100

实验目标：

1. 构建两个 ChampSim bin：
   - nopref-workload-sweep：不开 cache prefetcher，用于 baseline 和 trace 筛选
   - pref-workload-sweep：开启 prefetcher，用于和 nopref 做 IPC 对比

2. 遍历 trace 目录：
   - /data0/tzh/champsim_traces/SPEC06
   - /data0/tzh/champsim_traces/SPEC17
   - /data2/zcq/champsim_traces_gap
   - /data0/tzh/champsim_traces/Ligra
   - /data0/tzh/champsim_traces/QMM
   - /data0/tzh/champsim_traces/PARSEC
   - /data2/zcq/champsim_traces_xsbench

3. 先跑 nopref 全量 trace。

4. 用 nopref 结果筛选 trace：
   - 指标：Core_0_STLB_total_MPKI
   - 条件：Core_0_STLB_total_MPKI > 1.0

5. pref 只跑 nopref 筛选出的 selected trace。

pref 配置：

- L1D prefetcher：vberti
- L2C prefetcher：ip_stride

6. 默认只对筛选出的 trace 做后处理：
   - 每个配置各自输出 trace-level CSV
   - 每个配置各自输出 workload-level CSV
   - 每个配置各自画 STLB miss cause 百分比堆积柱状图
   - 对 pref vs nopref 做 IPC 对比
   - 对 pref vs nopref 做 STLB total miss rate 归一化对比

7. full_trace 后处理默认关闭：
   - 因为 pref 现在只跑 selected trace
   - 避免拿 nopref 全量 trace 和 pref selected trace 混在一起做 full_trace 对比

统计和聚合规则：

1. SPEC06 / SPEC17 / Ligra / PARSEC / XSBench：
   - 同一个 benchmark 下的多个 trace 作为同一个 workload
   - workload 内部 trace 等权聚合

2. GAP：
   - 使用 /data2/zcq/champsim_traces_gap
   - 按 bc / bfs / cc / pr / sssp / tc 聚合成六个 workload
   - 每个 workload 内部 trace 等权聚合

3. QMM：
   - 按 qmm_srv / qmm_int / qmm_fp 聚合成三个 workload

4. workload 内部：
   - IPC 使用 gmean
   - STLB MPKI 使用 trace-level MPKI 的 amean
   - STLB miss rate 使用 trace-level miss rate 的 amean
   - 各 miss-cause share 使用 trace-level share 的 amean

5. 最终汇总：
   - SPEC06 产生 gmean_spec06
   - SPEC17 产生 gmean_spec17
   - GAP 产生 gmean_gap
   - Ligra 产生 gmean_ligra
   - QMM 产生 gmean_qmm
   - PARSEC 产生 gmean_parsec
   - XSBench 产生 gmean_xsbench
   - 全部 selected trace 产生 gmean_all
   - dataset summary 按 DPC4-like 方式直接基于该 dataset 下所有 trace 计算
   - gmean_all 同样直接基于所有 selected trace 计算
   - IPC summary 使用 trace-level IPC 的 gmean
   - MPKI、miss rate、miss cause share summary 使用 trace-level 指标的 amean

============================================================
常用环境变量
============================================================

MAX_PARALLEL：
并行仿真数量，默认 4。

N_WARM：
warmup 指令数，单位是 million，默认 50。

N_SIM：
ROI 指令数，单位是 million，默认 100。

SKIP_EXISTING：
如果 result log 已存在且包含 [ROI Statistics] 是否跳过，默认 1。
已有 log 但没有完整 [ROI Statistics] 时，会自动重跑。

TRACE_FILTER：
只跑某一类 trace，例如：

TRACE_FILTER=649.fotonik3d_s
TRACE_FILTER=pr.

TRACE_DIRS：
trace 搜索目录，默认：

/data0/tzh/champsim_traces/SPEC06:/data0/tzh/champsim_traces/SPEC17:/data2/zcq/champsim_traces_gap:/data0/tzh/champsim_traces/Ligra:/data0/tzh/champsim_traces/QMM:/data0/tzh/champsim_traces/PARSEC:/data2/zcq/champsim_traces_xsbench

SELECT_THRESHOLD：
筛选 trace 的 STLB MPKI 阈值，默认 1.0。

SELECT_TRACE_JSON：
手动指定 selected trace JSON。默认使用：

csv_figure/spec0617_gapligra_qmm_parsec-TLB-select-trace-compare_vberti_50_100/select_trace/data_process_for_compare/stlb_mpki_gt_1.0_selected_traces.json
