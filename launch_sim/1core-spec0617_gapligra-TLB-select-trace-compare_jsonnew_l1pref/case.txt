筛选口径：

- baseline：旧实验 nopref
- 指标：Core_0_STLB_total_MPKI
- 条件：Core_0_STLB_total_MPKI > 1.0
- 当前 selected trace 数：43

本实验的 prefetcher 组合：

- nopref-workload-sweep：L1I/L1D/L2C/LLC 均 no，用于 baseline
- pref-workload-sweep：L1D=berti，L2C=pythia，原始 pref 配置
- pref-ipcp-pythia-workload-sweep：L1D=ipcp，L2C=pythia
- pref-berti-no-workload-sweep：L1D=berti，L2C=no
- pref-berti-ip_stride-workload-sweep：L1D=berti，L2C=ip_stride
- pref-sms-pythia-workload-sweep：L1D=sms，L2C=pythia

full_trace 流程已禁用。本目录只产出 select_trace 结果。

统计和聚合规则：

- SPEC06 / SPEC17 / Ligra：同一 benchmark 下多个 trace 聚为同一个 workload
- GAP：每个 trace 作为单独 workload
- workload 内 IPC 使用 trace-level IPC 的 geomean
- STLB MPKI 使用 sum(miss) * 1000 / sum(instructions)
- STLB miss rate 使用 sum(miss) / sum(access)
- DRAM read PKI 使用 sum(read_requests) * 1000 / sum(instructions)
- miss-cause/share 类指标使用 sum(cause) / sum(total)
- 数据集 summary 中 IPC 使用 geomean，MPKI/rate/share 仍使用 sum/sum

常用环境变量：

- MAX_PARALLEL：并行仿真数量，默认 4
- N_WARM：warmup 指令数，单位 million，默认 50
- N_SIM：ROI 指令数，单位 million，默认 200
- SKIP_EXISTING：已有 result log 是否跳过，默认 1
- TRACE_FILTER：在 selected trace 集合上再做 trace 名前缀过滤
- TRACE_DIRS：trace 搜索目录，默认 SPEC06/SPEC17/GAP/Ligra 四个目录
- SELECT_TRACE_JSON：覆盖默认复用的 selected trace JSON
- BASELINE_SELECT_TAG：覆盖默认 selected JSON 所在的旧实验 tag
