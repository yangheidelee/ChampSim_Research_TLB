launch_sim/1core-spec0617_gapligra-idealTLB-selectLLC-compare

实验目标：

1. baseline 使用 pref-workload-sweep。
   - 这是原始带 cache prefetcher 的配置。
   - 不开启 ideal STLB。

2. 构建并运行三种 ideal STLB 上限设计：
   - ideal-demand-workload-sweep：STLB 把 Demand Data 和 Demand Instruction 引发的 STLB miss 都当作真正 STLB hit 解决。
   - ideal-l1pref-workload-sweep：STLB 把 L1I Prefetch 和 L1D Prefetch 引发的 STLB miss 都当作真正 STLB hit 解决。
   - ideal-all-workload-sweep：STLB 把所有来源的 STLB miss 都当作真正 STLB hit 解决。

3. ideal hit 的含义：
   - 仍然走 STLB lookup。
   - 如果真实 tag lookup miss 且 miss 来源属于当前 ideal mode，则直接返回翻译结果。
   - 不进入 STLB miss 统计。
   - 不分配 STLB MSHR。
   - 不向 PTW/lower_translate 发送请求。
   - 不引入 page walk 延迟。
   - 在 STLB 统计中按 hit 计数。

4. 遍历两个 trace 目录：
   - /data0/tzh/champsim_traces/SPEC06
   - /data0/tzh/champsim_traces/SPEC17
   - /data0/tzh/champsim_traces/GAP
   - /data0/tzh/champsim_traces/Ligra

5. select_trace 流程：
   - 先用无预取 nopref-workload-sweep 的结果筛选 trace。
   - 筛选指标：Core_0_LLC_total_MPKI。
   - 默认条件：Core_0_LLC_total_MPKI > 1.0。
   - 后续 baseline 和三种 ideal 配置都只对筛选出的相同 trace 集合做后处理。

6. full_trace 流程：
   - 不做 LLC MPKI 筛选。
   - 直接使用 result 目录下所有可解析的 trace log。
   - 聚合公式和 select_trace 一致。

统计和聚合规则：

1. SPEC17：
   - 同一个 benchmark 下的多个 trace 作为同一个 workload。
   - workload 内部 trace 等权聚合。

2. GAP：
   - 每个 trace 作为单独 workload。

3. IPC：
   - trace-level IPC 直接来自 result。
   - workload-level IPC 使用同一 workload 内 trace IPC 的 gmean。
   - dataset summary IPC 使用筛选后所有 trace-level IPC 的 gmean。

4. IPC speedup：
   - trace-level speedup = ideal_trace_ipc / baseline_trace_ipc。
   - workload-level speedup = ideal_workload_ipc / baseline_workload_ipc。
   - dataset summary speedup = 所有 trace-level speedup 的 gmean。

============================================================
常用环境变量
============================================================

MAX_PARALLEL：
并行仿真数量，默认 4。

N_WARM：
warmup 指令数，单位是 million，默认 50。

N_SIM：
ROI 指令数，单位是 million，默认 200。

SKIP_EXISTING：
如果 result log 已存在是否跳过，默认 1。

TRACE_FILTER：
只跑某一类 trace，例如：

TRACE_FILTER=649.fotonik3d_s
TRACE_FILTER=gap.pr

TRACE_DIRS：
trace 搜索目录，默认：

/data0/tzh/champsim_traces/SPEC06:/data0/tzh/champsim_traces/SPEC17:/data0/tzh/champsim_traces/GAP:/data0/tzh/champsim_traces/Ligra

SELECT_THRESHOLD：
筛选 trace 的 LLC MPKI 阈值，默认 1.0。
